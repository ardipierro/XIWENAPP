   Currently: Native <input type="time"> in Create/Edit modal

   Status: KEEP AS-IS for UX consistency
           Hour/minute selects allow finer control for configuration

4. FORM VALIDATION
   Currently: Alert dialogs (alert(), confirm(), prompt())
   Status: Consider replacing with better UX dialogs
   Current Validation:
     - name required
     - schedules required (at least 1)
     - daysOfWeek required
     - startTime/endTime required
     - confirm before deletion
     - prompt for cancellation reason

5. DIALOG PATTERNS
   Create/Edit: alert() for errors
   Delete: confirm() dialog + prompt() for reason
   Status: Consider refactoring to use BaseModal for all dialogs
